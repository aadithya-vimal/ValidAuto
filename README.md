# AutoShield AI: Automated Vehicle Damage Assessment (Phase 5 Submission)

AutoShield AI is an end-to-end full-stack prototype designed to demonstrate computer vision damage classification and automated inspection report generation for vehicles. 

Built with **Next.js (App Router, Tailwind CSS v4)** on the frontend and **FastAPI (Python)** on the backend, the system allows users to upload exterior damage photos, classify damage types (Scratch, Dent, None), estimate severity tiers using model confidence thresholds, and generate insurance-ready PDF reports.

---

## 🛠️ System Architecture

The application is structured as a decoupled full-stack project:
1. **Frontend (Next.js + TypeScript)**: Renders a premium, dark-themed dashboard using glassmorphic panels and native print-screen interfaces. Includes a high-fidelity image scanner overlay and live classification HUD.
2. **Backend (FastAPI)**: Serves prediction endpoints and local report compilers.
3. **ML Classifier Engine (TensorFlow/Keras)**: Uses transfer learning (MobileNetV2 base feature extractor) to classify panel damage.
   * *Note on Compatibility*: For systems running Python 3.14 (where precompiled TensorFlow wheels are not yet available), a built-in emulated Keras layer executes model fits, data splits, and forward propagation using raw NumPy.

## 🧠 Machine Learning & Dataset Details

### 1. Data Preprocessing & Augmentation
* **Dimensions**: Input images are resized using PIL to $224 \times 224$ pixels to match the MobileNetV2 input layer layout.
* **Normalization**: Pixel values are normalized into the `[0.0, 1.0]` floating-point range.
* **Augmentation**: To generalize model training parameters, the training script uses PIL to apply:
  * Random horizontal flips (50% probability).
  * Random rotation within a range of $[-12^\circ, +12^\circ]$.
  * Random brightness adjustments ($[0.9, 1.1]$ factor range).

### 2. Model Structure (Transfer Learning)
The network relies on a MobileNetV2 base feature extractor initialized with ImageNet weights, topped with a custom classification layer:
* **GlobalAveragePooling2D**: Flattens spatial feature maps.
* **Dropout (0.2)**: Controls regularization during fitting.
* **Dense (128 units, ReLU)**: Fully connected feed-forward layer.
* **Dense (3 units, Softmax)**: Outputs category probabilities for `scratch`, `dent`, and `none`.

### 3. Model Performance Metrics
Upon running the training pipeline, the script evaluates the test split to compute:
* **Accuracy**: Overall prediction matching percentage.
* **Precision & Recall**: Classification boundary success metrics.
* **F1-Score**: Harmonic mean of Precision and Recall.
* **Confusion Matrix**: Visualizes correct classifications against mistakes.

Graphs for the **Training vs Validation Accuracy Curve** and the **Confusion Matrix Grid** are saved directly to `frontend/public/` and display on the **About** page.

## 🚀 Setup & Execution Guide

Open two terminal sessions to run both servers concurrently:

### Step 1: Run the Backend (FastAPI)
1. Navigate to the backend folder:
   ```bash
   cd backend
   ```
2. Activate the virtual environment:
   * **Windows Powershell**:
     ```powershell
     .\venv\Scripts\Activate.ps1
     ```
   * **Windows CMD**:
     ```cmd
     .\venv\Scripts\activate.bat
     ```
3. *(Optional)* Run the model training script:
   ```bash
   python app/train.py
   ```
4. Start the live FastAPI server:
   ```bash
   python run.py
   ```
   *Runs on `http://127.0.0.1:8000`.*

### Step 2: Run the Frontend (Next.js)
1. Navigate to the frontend folder:
   ```bash
   cd frontend
   ```
2. Install npm packages:
   ```bash
   npm install
   ```
3. Start the hot-reloading development server:
   ```bash
   npm run dev
   ```
   *Runs on `http://localhost:3000`.*
